# Security Policy

## Supported Versions

Security updates are provided for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly by following these steps:

### 1. Do Not Create Public Issues
Please **do not** report security vulnerabilities through public GitHub issues, discussions, or pull requests.

### 2. Report Privately
Send security reports to: [GitHub Security Advisories](https://github.com/MuhKar1/Counter-dApp/security/advisories/new)

Alternatively, you may email security concerns to the repository maintainer.

### 3. Include Details
When reporting a vulnerability, please include:
- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact and severity
- Any suggested fixes or mitigations

### 4. Response Timeline
- **Initial Response**: Within 48 hours
- **Vulnerability Assessment**: Within 7 days
- **Fix Development**: Within 30 days for critical issues
- **Public Disclosure**: After fix is deployed and tested

## Security Considerations

This project implements several security measures:
- Authority validation for all counter operations
- Integer overflow/underflow protection
- Program Derived Address (PDA) usage for secure account management
- Input validation and error handling

## Responsible Disclosure

We kindly ask that you:
- Allow reasonable time for fixes before public disclosure
- Avoid accessing or modifying user data
- Do not perform DoS attacks or degrade service performance

Thank you for helping keep this project and its users secure!
